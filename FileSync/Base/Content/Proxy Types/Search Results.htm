Search Results Proxy

The Search Results Proxy works with a Search Bar Proxy, and it creates a container that is used to display the results of end user searches in your output. By default, a search results page with the search results proxy is included in the HTML5 TopNav skin files. Creating your own topic with a Search Results proxy gives you more control over the text and look of the search results page.

If you add a Search Results proxy anywhere in your project, Flare will use that proxy for search results instead of its own default search results page. There is no need to add the search results topic to the TOC. Flare will pull it in automatically.

Do not place a Search Results Proxy in a MasterPage.

How the Search Results Proxy works in this project

When a search is performed from either the Search Bar Proxy page or the TopNav search bar, Flare returns the search results in the Search Results proxy in the Search Results Example.htm page.
