Menu Proxy

The Menu Proxy is used to generate a context-sensitive navigation menu in TopNav outputs. This menu typically is found to the left or right of the topic body content. It is placed in the MasterPage.

Viewing the Menu Proxy in this project source

Content Explorer > Resouces > MasterPages > default.flmsp

How the Menu Proxy works in this project

The default.flmsp MasterPage is assigned to the HTML5 target (see the Advanced tab of the target). When you build the HTML5 target, the default.flsmp MasterPage is applied to each topic, and the Flare TOC is built into a context-sensitive menu that replaces the Menu Proxy on each page in the HTML output.Learn more at http://help.madcapsoftware.com/flare2017/Content/Proxies/Using_the_Menu_Proxy.htm

In most projects, the menu proxy and body proxy are placed side-by-side. This requires some styles in the stylesheet to make this happen. In this project, the menu and body proxies are placed inside a responsive row that was created using Flare's Responsive Layout editor.Learn more at http://help.madcapsoftware.com/flare2017/Content/Interface/UI_Elements/Responsive_Layout_Window_Pane.htm

Try it! Inserting a Menu Proxy

Create a new MasterPage by right clicking Content Explorer > Resources > MasterPages and selecting New > MasterPage. Once the new MasterPage is in place, click Insert > Proxy > Insert Menu Proxy in the MasterPage. Note that the menu proxy and body proxy typically require some extra styling to get them to show up side-by-side.
