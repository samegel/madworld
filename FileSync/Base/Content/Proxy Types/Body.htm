Body Proxy

The Body Proxy is used as a topic placeholder in Flare MasterPages for online output. A MasterPage can contain only one Body Proxy. When you build your Flare HTML target, the content from your topics is inserted into the MasterPage where the Body proxy is.

Viewing the Body Proxy in this project source

Content Explorer > MasterPages > default.flmsp

In the MasterPage, it is labeled topic Body proxy.

How the Body Proxy works in this project

The default.flmsp MasterPage is assigned to the HTML5 target (see the Advanced tab of the target). When you build the HTML5 target, the default.flsmp MasterPage is applied to each topic, and the contents of each topic are inserted in place of the Body proxy.

Body Proxy Dependencies

Topic files with content in them.

Try it! Inserting a Body Proxy

Create a new MasterPage by right clicking Content Explorer > Resources > MasterPages and selecting New > MasterPage. Once the new MasterPage is in place, click Insert > Proxy > Insert Body Proxy in the MasterPage.
